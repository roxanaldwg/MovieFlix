Documentation

The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Technical Requirements

  • The API must be a Node.js and Express application;
  • The API must use REST architecture, with URL endpoints corresponding to the data operations listed above;
  • The API must use at least three middleware modules, such as the body-parser package for reading data from requests and morgan for logging;
  • The API must use a “package.json” file;
  • The database must be built using MongoDB;
  • The business logic must be modeled with Mongoose;
  • The API must provide movie information in JSON format;
  • The JavaScript code must be error-free;
  • The API must be tested in Postman;
  • The API must include user authentication and authorization code;
  • The API must include data validation logic;
  • The API must meet data security regulations;
  • The API source code must be deployed to a publicly accessible platform like GitHub;
  • The API must be deployed to Heroku.

Essential Features

Business Logic URL HTTP Method Response body data format
Home page MovieFlix / GET Message: "Welcome to MovieFlix!
Return a list of ALL movies to the user /movies GET A JSON object holding data about all the movies
Return data about a single movie by title to the user /movies/:Title GET A JSON object holding data about a single movie, containing description, genre, director, image URL, whether it’s featured or not.
Example:
{
"id": "...",
"title": "...",
"description": "...",
"genre": "...",
"director": "...",
"imagePath": "...",
"featured": "..."
}
Return data about a genre /genres/:name GET Return JSON object with data about genre (description) by name/title (e.g., “Thriller”)
Example response
{
"title": "...",
"genre": "...",
}
Return data about a director by name /directors/:name GET Return JSON object with data about director (bio, birth year, death year) by name
Example response
{
"director": "...",
"name": "...",
"bio": "...",
"birth": "...",
"death": "..."
}
Allow new users to register /users POST Create JSON object with data about a new user.
Example response
{
"id": "...",
"username": "...",
"password": "...",
"email": "...",
"birthday": "...",
"favoriteMovies": "..."
}
Allow users to update their user info /users/:username PUT Update JSON object with user data.
Example
{
"id": "...",
"username": "...",
"password": "...",
"email": "...",
"birthday": "...",
"favoriteMovies": "..."
}
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added /users/:username/movies/:movieID POST Create JSON object with data about a user's list of favorite movies. Message: "Movie was added to favorites"
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed) /users/:username/movies/:movieID DELETE JSON object with data about a user's list of favorite movies. Message: "Movie was deleted"
Allow existing users to deregister (showing only a text that a user email has been removed) /users/:username DELETE Delete user's account. Message: "Your account was successfully deleted"
